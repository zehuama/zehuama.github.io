4.1 建立Post的Model

目标

在这一章，我们要带大家在群里面可以发文章。我们会完成以下事项：

  • 文章 Post 要有“内容”、“所属讨论群组”、“叙述”
  • 发文网址是在 http://localhost:3000/groups/1/posts/new

步骤

Step 0. git checkout -b ch04

因为我们现在要实作第四章的内容，新增一个 branch 把实作内容集中在 ch04 是比较合理的。将来回顾就可以比较清楚 在 ch04 我们做了哪一些练习。

git checkout -b ch04

Step 1 : 建立文章 Post 的 Model

  • 文章要有「內容」
    所以我們在這裡要建立一個 model: Post（文章）。文章內容必須要有 content （內容），並且要紀錄屬於哪個群組 group_id

  • 建立 Model: post
    便建立資料表 post 的二個欄位: content(text 文字屬性) 跟 group_id (integer 整數屬性)

    rails g model post content:text group_id:integer

  • 將 post 的資料庫建立起來
    rake db:migrate

Step 2: 建立 Group 與 Post 之間的關係

一個看板 Group 可以有很多文章

一篇文章 Post 可以屬於 Group

  • git 存档
    git add .
    git commit -m "add post relation with group"

Step 3 : 建立雙層發文網址 /groups/1/posts/2

建立 Controller: posts
rails g controller posts

建立雙層網址
因為我們最後目的是想要建立 http://localhost:3000/groups/1/posts/2 這樣的雙層網址。這樣的網址在 Rails 是屬於「巢狀」機制。

修改 config/routes.rb

之後我們對於 Post 的 CRUD 操作都會在 http://localhost:3000/groups/1/posts 下的這一層了。

  • 做了这件事后，执行
    rake routes

然后你就会发现， routing 列表内产生了 /groups/:group_id/posts/new 这样的网址支援。

添加logdown

    git commit -m "add posts nested route"

目標

實作
